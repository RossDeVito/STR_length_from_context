# GPU Python deps for the Caduceus-Ph stack. Installed by setup_gpu_env.sh.
#
# Uses the Caduceus authors' tested kernel triple (from their caduceus_env.yml):
#   torch 2.2.0 + mamba-ssm 1.2.0.post1 + causal-conv1d 1.2.0.post2
# Those mamba/causal wheels are built against torch 2.2.0 EXACTLY, so there is no
# ABI/symbol drift (the recurring failure when matching newer-torch wheels).
# Prebuilt: no CUDA module, no compiling. cu118 runs fine on the modern driver.
#
# The mamba/causal kernel wheels themselves are installed by setup_gpu_env.sh
# (cp311 / cu118 / torch2.2 / cxx11abiFALSE), not listed here.
#
# torch 2.2 predates numpy-2 / pandas-3, so those are capped; pip resolves the
# rest of the science stack to numpy-1.26-compatible versions. Use a FRESH
# python 3.11 env (this changes torch 2.x and CUDA 12->11).

# --- PyTorch (CUDA 11.8 build) ---
--extra-index-url https://download.pytorch.org/whl/cu118
torch==2.2.0+cu118

# --- transformers 4.x (has the generation symbols mamba 1.2 imports) ---
transformers==4.44.2

# --- caps required by torch 2.2 ---
numpy<2
pandas<3

# --- Everything else (lightning, captum, pyfaidx, tensorboard, ...) ---
-r requirements.txt
