# GPU Python deps for the Caduceus-Ph stack. Installed by setup_gpu_env.sh.
#
# Uses PREBUILT mamba-ssm + causal-conv1d wheels (no compiling, no CUDA module).
# torch 2.6 is used because BOTH mamba-ssm and causal-conv1d publish matched,
# fused-capable cu12torch2.6 wheels in both C++ ABIs — so a wheel always matches
# torch's ABI. (torch 2.5 had no abiFALSE causal wheel with the cpp_functions API,
# which is what forced the painful source-build path; torch 2.6 avoids that.)
#
# The kernel wheels are installed by setup_gpu_env.sh (their URL depends on
# torch's detected ABI), not listed here.
#
# Assumes Python 3.11 and a driver supporting CUDA 12.4 (TSCC A100 / driver 580 does).

# --- PyTorch (CUDA 12.4 build) ---
--extra-index-url https://download.pytorch.org/whl/cu124
torch==2.6.0+cu124

# --- transformers 4.x ---
# The Caduceus trust_remote_code modeling/tokenizer targets ~4.38 and isn't
# updated for transformers 5.x; 4.44.2 also supports numpy 2.x.
transformers==4.44.2

# --- Everything else (numpy, pandas, lightning, captum, ...) ---
-r requirements.txt
